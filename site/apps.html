Every app here is a wasi:http component whose bytes live on IPFS, addressed by their CID. The catalog itself is a smart contract on Base (NanAppCatalog): anyone can publish, each listing is attributed on-chain to the publisher's wallet, and because a CID is a hash of the exact wasm you can pull it from any IPFS peer and verify the bytes yourself. Reading needs no wallet; publishing is one Base transaction.

Publish a Wasm app version

Add a wasi:http component release to the on-chain catalog. Its bytes go on IPFS; the CID and metadata go on Base. An app is keyed by slug in your wallet's namespace; publish the same slug again to cut a new version. Only you (msg.sender) can add versions, edit, or delist it. New to building components? The developer guide takes you from an empty directory to a published app.
