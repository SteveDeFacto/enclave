# vault — the web UI for wallet-gated encrypted volumes

A `wasi:http` component that serves the browser side of nan-vault (Phase 4 of
the encrypted-volumes design). Users connect a wallet, derive their vault key
(one deterministic `personal_sign` — see `DERIVE_MESSAGE` in
`scripts/nan-vault.mjs`), self-register on the NanVolumeAccess ACL, and unlock
deployments by sealing the volume key to an **attestation-verified** enclave.
Owners create volumes, bootstrap the VEK, grant/revoke members.

The page is fully self-contained: `vault.js` vendors the protocol
(`scripts/nan-vault.mjs`), viem and `@tinfoilsh/verifier` at build time.
Nothing loads from a CDN at runtime — a runtime-fetched script could
exfiltrate the wallet-derived key or an unsealed VEK.

## Routes

| route          | what                                                       |
|----------------|------------------------------------------------------------|
| `GET /`        | the app                                                    |
| `GET /vault.js`| the vendored bundle (generated by `js/build.mjs`, checked in) |
| `GET /config.json` | embedded defaults ⊕ `NAN_CONFIG` (chain, contracts, repo) |
| `GET /ping`    | liveness                                                   |

Deep links: `/?owner=0x…&volume=user-data&dep=0x…` (plus `contract=`, `rpc=`,
`deployments=`, `registry=`, `repo=`, `chain=` overrides).

## Trust chain (what the page enforces, in order)

1. **Discovery from chain state** — `NanDeployments.get(id).runner` is
   `keccak256` of the runner's NanRegistry endpoint; no gateway is trusted to
   say where a deployment runs.
2. **Attestation before key release** — `@tinfoilsh/verifier` verifies the
   runner origin measures as its registry `repo`; only then is the enclave's
   per-boot vault pubkey fetched over that TLS (deployment-scoped path) and
   the VEK sealed to it.
3. **On-chain ACL** — SIWE only names the member; the enclave gates
   `unlock-sealed` on `isAuthorized(volId, member)`.

Key material (wallet-derived X25519 secret, unsealed VEK) lives in page RAM
only; "Forget key material" zeroes it, reload forgets everything.

## Build

```sh
# 1. the JS bundle (only when js/vault-app.mjs changes; output is checked in)
node wasm/apps/vault/js/build.mjs

# 2. the component
cd wasm/apps/vault && cargo component build --release --target wasm32-wasip2

# 3. sanity (the platform always passes -Scli -Shttp)
wasmtime serve --addr 127.0.0.1:8377 -S cli target/wasm32-wasip*/release/vault.wasm
node wasm/apps/vault/js/test-bundle.mjs   # protocol interop vs scripts/nan-vault.mjs
```

`config.json` defaults pin NanDeployments/NanRegistry on Base;
`volumeAccess` is filled at go-live (hand-deployed contract) and can always be
overridden per deployment via `NAN_CONFIG` (configCid) or `?contract=`.

## Publish

Pure `wasi:http`, no GPU, no ports beyond the served app, minimal memory —
publish like `hello`. Pairs with the always-on unlock agent:
`node scripts/nan-vault-client.mjs watch --id 0x… --owner 0x… --name <vol>`
(re-delivers the sealed VEK on failover/restart; detects fresh enclave boots
by their per-boot vault pubkey).
